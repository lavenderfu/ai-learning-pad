# 阶段3：工程交付 协议

## 角色职责

- **工程师（主导）**：页面打磨、交互完善、联调部署、最终交付
- **架构师（辅助）**：接口调整
- **PM（辅助）**：最终验收

## 进入条件

先执行阶段2完成度检查：
1. 读取 `2-build/docs/handover.md`，了解完成情况和遗留项
2. 读取项目级 `2-build/CLAUDE.md`，了解技术栈和代码规范
3. 尝试运行项目，确认可启动
4. 如有缺失，列出问题并提醒用户，**不开始开发**

## 工作流程

### 1. 审阅代码

Review 阶段2的代码，评估质量和结构：
- 代码结构是否清晰
- 命名是否规范
- 是否有明显的代码异味
- 安全隐患检查

### 2. 列出打磨清单

根据 handover.md 和自己的审查，列出所有待办：
- handover.md 中的遗留项
- 代码审查发现的问题
- UI/交互需要改进的地方
- 性能优化点

### 3. UI/交互打磨

- 样式精修：间距、字体、颜色等视觉细节
- 动效：过渡动画、loading 动效
- 响应式适配：移动端/平板/桌面端

### 4. 联调

- 对接真实接口，替换 mock 数据
- 验证数据流转正确性
- 处理接口异常场景

### 5. 异常处理

- 加载状态：骨架屏、loading 动画
- 空状态：无数据时的友好提示
- 错误提示：网络错误、服务端错误的用户友好提示
- 边界情况：超长文本、特殊字符等

### 6. 部署上线

- 构建生产版本
- 配置生产环境变量
- 部署到目标平台
- 验证线上功能
