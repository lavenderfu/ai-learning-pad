# 阶段2：核心搭建 交接文档

## 1. 已完成的功能 (Done)
- [x] **项目框架搭建**：基于 Next.js + Tailwind CSS 搭建。
- [x] **家长台组件 (`ParentDashboard`)**：包含可视化数据面板，以及输入配置主题的交互。
- [x] **防误触学习台 (`ChildLearning`)**：全屏沉浸、无返回按键、防误触动画反馈机制（震动报错，星星庆祝）。
- [x] **防误触长按解锁**：小锁图标长按 1.5s（为方便演示缩小了时间）唤起数学题解锁模态框。
- [x] **AI 动态题库代理 (`/api/generate-questions`)**：集成了 `@ai-sdk/anthropic`，能够根据家长输入的文本动态生成结构化的题目 JSON（3道题）。
- [x] **前后端联调**：学习页面进入时展现 Loading，动态请求后端 AI 生成题目并渲染交互。包含降级/报错后的容错 Mock 处理。

## 2. 遗留问题与技术债 (Tech Debt)
1.  **语音合成 (TTS)**：当前仅有 UI 上的播放动效反馈，未真正引入浏览器原生 Speech API 或第三方 TTS 来实现声音播报。由于浏览器策略，后续工程师阶段需要处理用户在播放音频前先进行首次交互（比如点击“开始学习”按钮作为触发点）。
2.  **云端真实存储**：目前家长看板右侧提示的“已同步来自云端远程配置”只是 Mock 态，实际上并未接入真实数据库（如 Supabase）。
3.  **拖拽题型**：当前完成了“听音点按卡片”题型，由于时间所限，“具象化拖拽题型 (`dnd-kit`)” 尚未深入开发，可作为迭代二期的组件。

## 3. 工程师阶段建议 (For Stage 3)
- 测试 AI 接口时，请在 `.env.local` 根目录配置真实的 `ANTHROPIC_API_KEY`。
- 如果不配置 API Key，应用会自动平滑降级（使用内置的一道 Mock 题展示 UI，这是设计好的容错机制）。
