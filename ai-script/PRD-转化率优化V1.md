# AI漫剧大师 - 转化率优化需求文档 V1

## 一、背景与目标

### 1.1 现状问题
- 腾讯广告投放，2天曝光 1000+ 次
- 功能使用（点击开始创作）仅 2-3 次
- 转化率约 0.2-0.3%

### 1.2 问题分析
- 广告流量用户期待看到「成品效果」，落地页却要求「主动输入」
- 创意示例在首屏下方，用户看不到就离开
- 从 0 开始写故事的门槛过高

### 1.3 优化目标
- 提升广告流量 → 功能使用的转化率
- 目标：转化率提升至 10%-20%

---

## 二、整体方案

### 2.1 核心策略
从「功能驱动转化」改为「内容驱动转化」：

```
原路径：看功能介绍 → 自己想内容 → 创作（门槛高）
新路径：看爆款剧本 → 产生兴趣 → 生成我的剧本（门槛低）
```

### 2.2 结构调整

| 原结构 | 新结构 |
|-------|-------|
| 工作台 / 历史项目 / 我的 | **爆款剧本** / 工作台 / 我的 |

- **爆款剧本**：推荐流（广告流量默认落地页）
- **工作台**：输入创作 + 历史项目入口
- **我的**：个人中心（创作点、明细、设置等）

## 三、功能详细设计

### 3.1 Tab 导航栏

#### 结构
```
[ 爆款剧本 ]    [ 工作台 ]    [ 我的 ]
```

#### 规则
- 广告渠道进入：默认落地「爆款剧本」Tab
- 自然流量 / 回访用户：默认落地「工作台」Tab
- 判断逻辑：通过广告链接携带参数 `source=ad` 区分

---

### 3.2 爆款剧本页（新增）

#### 3.2.1 页面结构

```
┌─────────────────────────────────────┐
│  顶部引导条                          │
├─────────────────────────────────────┤
│  剧本卡片    │    剧本卡片           │
│  剧本卡片    │    剧本卡片           │
│  ...        │    ...               │
│         (双列瀑布流)                 │
└─────────────────────────────────────┘
```

#### 3.2.2 顶部引导条

- **文案**：「🔥 10w+人正在用AI生成爆款漫剧，点击剧本立即体验」
- **样式**：渐变背景（主色渐变，复用 --primary-gradient），白色文字，左侧动态火焰图标，高度 48px，字号 14px 加粗
- **动效**：入场时从上方滑入（0.3s），文案呼吸灯效果（透明度 0.85 ↔ 1.0 循环）
- **交互**：常驻展示，不可关闭；点击引导条 → 自动滚动到第一个剧本卡片

#### 3.2.3 剧本卡片

**卡片内容：**
- 封面图（9 宫格缩略图，复用现有样式）
- 剧本标题（最多 10 字，超出截断）
- 标签：集数（如「10集」）
- 热度标识（可选，如「🔥 1.2w 人看过」）

**卡片样式：**
- 双列瀑布流布局
- 卡片圆角 12px
- 卡片间距 12px
- 点击态：轻微缩放 0.98

**交互：**
- 点击卡片 → 进入剧本阅读页

#### 3.2.4 数据要求

- 运营配置 20 条爆款剧本
- 支持后台配置排序权重
- 支持上下架控制

---

### 3.3 剧本阅读页（新增）

#### 3.3.1 页面结构

```
┌─────────────────────────────────────┐
│  ←  剧本标题                         │  ← 顶部导航
├─────────────────────────────────────┤
│                                     │
│           剧本正文内容               │
│       (分集展示，可滚动，不可复制)     │
│                                     │
├─────────────────────────────────────┤
│  [ 下载该剧本 ]   [  我也做一个  ]   │  ← 底部操作栏（在 Tab 栏之上）
├─────────────────────────────────────┤
│  [ 爆款剧本 ]  [ 工作台 ]  [ 我的 ] │  ← 底部 Tab 栏
└─────────────────────────────────────┘
```

#### 3.3.2 顶部导航栏

- 左侧：返回按钮
- 中间：剧本标题（最多显示 12 字）

#### 3.3.3 剧本正文区

**展示方式：**
- 按集分段展示，每集有明显分隔
- 集标题样式：「第 1 集」加粗居中
- 正文：标准阅读排版（两端对齐），行高 2.0，字号 17px，段落间距加大（60px）

**阅读交互：**
- 支持上下滚动阅读全部内容（单页长滚动）
- 底部显示当前阅读进度：「共 10 集」
- 剧本正文内容不可复制（禁用文本选择）

**阅读体验：**
- 背景色：#F8F6F0（护眼阅读纸张色）

> 💡 **目录功能暂不做**：本期内容量较小，单页滚动足够。后续如内容扩展或用户有跳集需求，二期考虑增加目录导航。

#### 3.3.4 底部操作栏

**位置：**
- 固定在底部 Tab 栏之上，不与 Tab 栏重合
- 底部安全距离：适配 iPhone 底部安全区（由 Tab 栏处理）

**布局：**
- 左右双按钮布局，间距 12px
- 操作栏内边距：左右各 20px，上下各 8px

**左侧按钮 —「下载该剧本」：**
- 主文案：「下载该剧本」
- 副文案（小字）：根据用户状态变化
  - 成功项目数 > 0：**不展示副文案**（主文案垂直居中）
  - 成功项目数 = 0：「🔒 成功创作1次解锁」（锁图标 + 提示文字）
- 样式：
  - 解锁状态：白色背景，主色边框，主色文字，正常可点击
  - 未解锁状态：维持主色线框样式，但透明度降低（opacity: 0.7），点击时提示需要先创作
- 宽度：占剩余空间 50%
- 高度：40px
- 圆角：20px

**右侧按钮 —「我也做一个」：**
- 文案：「我也做一个」
- 样式：主色渐变背景（复用现有 --primary-gradient），白色文字
- 宽度：占剩余空间 50%
- 高度：40px
- 圆角：20px

**交互：**
- 「下载该剧本」→ 当满足解锁条件（历史项目>0）时点击，直接下载该剧本（不再消耗创作点）。未满足条件时点击会弹出 Toast 提示：“需要成功创作1次项目后才能解锁下载权益”。
- 「我也做一个」→ 跳转工作台页（带参数）

**高亮提示：**
- 触发条件：进入剧本阅读页后自动触发并持续展示
- 表现：「我也做一个」按钮呼吸灯式放大（scale 1.0 ↔ 1.05）+ 蓝色外发光效果，循环播放（infinite）
- 触发次数：页面停留期间持续触发

### 3.4 工作台页（改造）

> **改造原则**：调整工作台样式，使其更精简，同时在底部新增「历史项目」入口。

#### 3.4.1 页面结构与样式调整

1. **移除风格选项**：删除输入框下方的“风格”下拉选择框，仅保留“集数”选择框。
2. **移除创意示例**：删除工作台原有的“创意示例”双列卡片区域（因与新增的爆款剧本内容重复）。
3. **新增历史入口**：在开始创作按钮下方新增历史项目入口：

```
┌─────────────────────────────────────┐
│  [ 集数选择 ]                        │
│  [     开始创作按钮     ]            │
├─────────────────────────────────────┤
│  历史项目 (3)                     →  │  ← 新增入口
└─────────────────────────────────────┘
```

**位置：** 工作台底部，开始创作按钮下方

**样式：**
```
┌─────────────────────────────────────┐
│  📁 历史项目 (3)                  →  │
└─────────────────────────────────────┘
```

**规则：**
- 括号内数字 = 用户历史项目数量
- 常驻展示，不隐藏
- 数量为 0 时：显示「历史项目 (0)」，置灰不可点击
- 数量 > 0 时：点击 → 进入历史项目列表页（复用现有页面）

## 六、技术要点

### 6.1 广告落地页判断

```javascript
// 从广告链接获取参数
const source = getQueryParam('source');
if (source === 'ad') {
  // 默认展示「爆款剧本」Tab
  switchTab('discover');
} else {
  // 默认展示「工作台」Tab
  switchTab('workbench');
}
```

### 6.2 阅读页按钮逻辑

```javascript
// 点击「下载该剧本」按钮
function onDownloadClick(scriptId) {
  // 判断历史项目数量，如果大于0则允许下载（无消耗）
  if (historyData.length > 0) {
    downloadScript(scriptId);
  }
}

// 点击「我也做一个」按钮
navigateTo('/pages/workbench', {
  from: 'script_read',
  script_id: '123'  // 用于埋点追踪来源
});
```

无需参数联动，工作台保持默认状态。

## 八、后续迭代方向（本期不做）

1. **AI 灵感生成**：用户不知道写什么时，AI 随机生成故事梗概
2. **分享裂变**：阅读页分享功能，邀请好友得创作点
3. **用户 UGC**：优质用户作品进入推荐流（需授权机制）
4. **个性化推荐**：基于用户行为推荐感兴趣的剧本风格
